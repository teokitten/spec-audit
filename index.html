OpenAPI Spec Audit

Get a scored report of documentation gaps in an OpenAPI 3.0/3.1 spec – missing descriptions, absent examples, undocumented error responses, loose type constraints, semantic inconsistencies, and spec-wide naming/description consistency. Configure checks, export reports, and compare runs over time.

This tool audits documentation quality, not schema correctness; it doesn't render your spec as reference docs.

Overall score: the average of every endpoint's weighted score. Checks that block a developer completely (missing descriptions, undocumented errors) count twice as much as checks that just slow them down (missing examples, loose constraints, ambiguous required fields).

  • A = 90–100
  • B = 75–89
  • C = 60–74
  • D = 40–59
  • F = 0–39

This is a documentation-quality heuristic, not a spec validator – it checks whether a reader has what they need, not schema correctness. Checks turned off in "Configure checks" are excluded from scoring entirely, not just hidden from the report.

Known limitations
  • Short descriptions: brief but correct text (e.g. "Not Found," 9 characters) may still get flagged just for being under 10 characters.
  • Field-name mentions: looks for field names in text – easily confused by lists of allowed values, and therefore off by default.
  • Same word, different meaning: a term used inconsistently on purpose vs. by mistake isn't always distinguishable.
