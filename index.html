OpenAPI Spec Audit

Paste an OpenAPI 3.0/3.1 spec to get a scored report of its documentation gaps – missing descriptions, absent examples, undocumented error responses, and loose type constraints.

This tool audits documentation quality, not schema correctness; it doesn't render your spec as reference docs.

  • Grade = average endpoint score:

  • A missing description or undocumented error costs an endpoint twice as many points as a missing example, loose constraint, or ambiguous required field.

This is a documentation-quality heuristic, not a spec validator – it checks whether a reader has what they need, not schema correctness. Short but accurate descriptions (e.g. standard HTTP status text) may still be flagged for review. Checks turned off in "Configure checks" are excluded from scoring entirely, not just hidden from the report.
